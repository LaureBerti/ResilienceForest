## Prerequisites

### Python

`stumpy` (used in E2–E8, E12) requires `numba`/`llvmlite`, which have no pre-built wheels for Python 3.13.  
**Use Python 3.12 for all experiments.**

```bash
pyenv install 3.12.9
pyenv local 3.12.9
```

E1 data-prep scripts and figure-generation scripts run on Python 3.13 (no stumpy dependency).

### macOS 15+ (Darwin 25) — llvmlite from source

On macOS Sequoia (Darwin 25+), PyPI has no pre-built `llvmlite` wheel. Build it against LLVM 20:

```bash
brew install llvm@20
CMAKE_PREFIX_PATH=$(brew --prefix llvm@20) \
LLVM_CONFIG=$(brew --prefix llvm@20)/bin/llvm-config \
pip install llvmlite==0.47.0
```

Then proceed with the full `requirements_experiments.txt` install below.

### HDF4 C library (required only for `exp_E1_earthaccess.py`)

```bash
conda install -c conda-forge hdf4
INCLUDE_DIRS=$(conda info --base)/include \
LIBRARY_DIRS=$(conda info --base)/lib \
pip install pyhdf --no-binary pyhdf
export DYLD_FALLBACK_LIBRARY_PATH=$(conda info --base)/lib   # add to .zshrc
```

## Data preparation (E1)

All datasets must be prepared before running experiments E2–E14.

### IFL boundaries (manual download)

Download the Intact Forest Landscape GeoPackages from https://www.intactforests.org/data.ifl.html  
and place them in `data/raw/ifl/`:

```
data/raw/ifl/IFL_2000.gpkg
data/raw/ifl/IFL_2013.gpkg
data/raw/ifl/IFL_2016.gpkg
data/raw/ifl/IFL_2020.gpkg
data/raw/ifl/IFL_2025.gpkg
```

Convert to GeoParquet (one-time, ~5 min):

```bash
python experiments/exp_E1_convert_ifl_gpkg.py
```

### Climate indices (auto-download from NOAA CPC)

```bash
python experiments/exp_E1_climate_download.py
```

Outputs `data/raw/climate/`: AO, NAO, PNA, ENSO monthly CSVs + `climate_indicators_2000_2025.csv`.

### MODIS MOD13A3 kNDVI — Option A: NASA Earthdata (recommended)

Requires a free NASA Earthdata account: https://urs.earthdata.nasa.gov

```bash
# First run — cache credentials interactively
python -c "import earthaccess; earthaccess.login(persist=True)"

# Full download (14 regions, 2000–2025; ~50–80 GB HDF4 tiles)
python experiments/exp_E1_earthaccess.py

# Specific regions or IFL years
python experiments/exp_E1_earthaccess.py --regions amazon congo --ifl-years 2020
```

Output: `data/raw/kndvi/kndvi_{region}_2000_2025.parquet`

| IFL vintage | MODIS period applied |
|-------------|---------------------|
| IFL 2000    | 2000-01 – 2012-12   |
| IFL 2013    | 2013-01 – 2015-12   |
| IFL 2016    | 2016-01 – 2019-12   |
| IFL 2020    | 2020-01 – 2024-12   |
| IFL 2025    | 2025-01 – 2025-12   |

### MODIS kNDVI — Option B: Google Earth Engine

1. Open https://code.earthengine.google.com/ and paste `experiments/exp_E1_gee.js`
2. Set `var IFL_YEAR = 2020` and `var MVP_MODE = false`; click **Run**
3. Download CSVs from Google Drive after tasks complete
4. Add your Drive file IDs to `exp_E1_convert_to_parquet.py` and run:

```bash
python experiments/exp_E1_convert_to_parquet.py
```

## Running experiments

Run in this dependency order:

```
E1 (data prep)
  └─► E2 (matrix profiles)
        └─► E3 (motif–climate overlap)
              └─► E5 (temporal shift test)
              └─► E10 (FDR correction — after E5 and E6)
              └─► E11 (overlap null model)
              └─► E12 (m-sensitivity)
        └─► E4 (baselines — parallel with E3)
  └─► E6 (rolling correlation)
        └─► E13 (Hamed-Rao corrected MK — after E6)
  └─► E7 (structural changepoints)
  └─► E8 (motif predictability index)
  └─► E9 (pre/post-2020 trend)
  └─► E14 (E5 floor analysis — after E5)
```

### Shell runners (E1–E8)

```bash
source .venv_experiments/bin/activate
bash experiments/run_exp_E1.sh
bash experiments/run_exp_E2.sh
bash experiments/run_exp_E3.sh
bash experiments/run_exp_E4.sh
bash experiments/run_exp_E5.sh
bash experiments/run_exp_E6.sh
bash experiments/run_exp_E7.sh
bash experiments/run_exp_E8.sh
```

### Direct Python invocation (E9–E14)

```bash
source .venv_experiments/bin/activate

python experiments/exp_E9_recent_trend.py
python experiments/exp_E10_fdr_correction.py       # requires E5 + E6 outputs
python experiments/exp_E11_overlap_null_model.py   # requires E3 outputs
python experiments/exp_E12_m_sensitivity.py        # requires raw kNDVI + climate
python experiments/exp_E13_hamed_rao_mk.py         # requires E6 outputs
python experiments/exp_E14_e5_floor_analysis.py    # requires E5 outputs
```


### Hydra config overrides

```bash
# Change subsequence length (default m=24)
python experiments/exp_E2_matrix_profile.py matrix_profile.subsequence_length=12

# Change FDR level (default q=0.05)
python experiments/exp_E10_fdr_correction.py analysis.alpha=0.01

# Sweep m values for sensitivity analysis
python experiments/exp_E12_m_sensitivity.py analysis.m_values=[12,24,36,48]

# Dump default config
python experiments/exp_E2_matrix_profile.py --write-config
```

All parameters are defined in `conf/resilience_forest.yaml`.

## Known issues

- **GEE exports** are asynchronous; total time depends on the GEE queue (30 min – several hours for all 14 regions).
- **`pycwt` (E4)** may require 4+ GB RAM for 14 regions × 288 months. Process one region at a time: `python experiments/exp_E4_baselines.py regions.continents=[africa]`
- **`run_mvp_only=true`** uses synthetic stand-ins for missing data — results are NOT valid for paper figures.
- **`cartopy`** is optional; map figures fall back to a geopandas plain-axes render if not installed.
